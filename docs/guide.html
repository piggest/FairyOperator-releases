機能ガイド

FairyOperator の全機能をひとつのページにまとめた。妖精ごとに「できること」「設定項目」「ホットキー」「必要な権限」を確認できる。

前提条件

対応 macOS13.0 (Ventura) 以降
起動形態メニューバー常駐（Dock アイコンなし）
必須権限アクセシビリティ／入力モニタリング（ほとんどの妖精で必要）
追加権限画面収録（ウインドウサムネイル取得時）

初回セットアップ — システム設定 ›「プライバシーとセキュリティ」› アクセシビリティ と 入力監視 に FairyOperator を追加すると、ほとんどの妖精が動き始める。

キー入れ替えの妖精

接続中のキーボードごとにキーをリマップ。デバイス別プロファイルで管理。

主な機能

  • キーボードデバイスごとに独立したマッピングプロファイル
  • 通常キー → 通常キーのリマップ（CGEvent タップ）
  • 修飾キー同士のリマップ（例: CapsLock → Control）
  • CapsLock → 通常キーのリマップ
  • HID-only キー（CGEvent に現れない特殊キー）のリマップ
  • 未接続キーボードのプロファイル保持と再接続時の自動復元
  • 最後に押されたキーコードのリアルタイム表示

設定項目

  • 有効トグル（全体 ON/OFF）
  • キーボード一覧（接続中・未接続を区別。右クリックでプロファイル削除）
  • マッピングエディタ（from → to のペア、ペアごとの有効/無効、プロファイル名編集）

必要な権限

アクセシビリティ／入力モニタリング

使用例

  • 日本語キーボードの「英数」「かな」を Escape など他キーに割り当てる
  • CapsLock を Control にする

合成イベントには内部マーカー FAIRY を付加して二重リマップを防止している。修飾キー同士のリマップは macOS の HID プロパティを利用するため、アプリ終了後に一時的に残ることがある。

アプリ切り替えの妖精

ホットキーで最近使用順のアプリ一覧を呼び出す。Cmd+Tab の代替。

主な機能

  • 最近使用順に並んだアプリ一覧パネル（最大 7 列グリッド）
  • ホットキー押し続けで前後選択、離した瞬間に確定
  • Escape でキャンセル、マウスホバー＋クリックでも確定
  • 右クリックでウインドウピッカー（サムネイル付き、複数ウインドウから個別選択）
  • 最小化ウインドウの自動復元
  • Unity Editor はプロジェクト固有アイコンを自動表示
  • 使用順を再起動後も保持

設定項目

  • 有効トグル
  • 進む／戻るキーバインド

ホットキー

操作デフォルト
進む⌥Tab
戻る⌥⇧Tab
キャンセルEsc
ウインドウピッカー右クリック

必要な権限

アクセシビリティ（ウインドウ復元）／画面収録（ウインドウサムネイル）

使用例

  • Cmd+Tab より少ない操作でアプリ切り替え
  • 多ウインドウアプリの特定ウインドウを直指定

クリップボードを集める妖精

コピー履歴を自動蓄積。ホットキーから過去のテキスト・画像を貼り付け。

主な機能

  • 0.5 秒ポーリングでクリップボード変更を検出
  • テキスト／画像（PNG・TIFF）の両方を保持
  • パスワードマネージャー等の機密タイプ（org.nspasteboard.ConcealedType）は自動除外
  • ピン留めで先頭固定、自動削除対象から除外
  • インクリメンタル検索
  • 数字キー 1〜9 で直接選択
  • 確定後は前のアプリに自動復帰して ⌘V 送出
  • 選んだアイテムは先頭に再配置（ピン留め直後）

設定項目

  • 有効トグル
  • 呼び出しホットキー（デフォルト: ⌃V）
  • 検索ホットキー（デフォルト: ⌘F）
  • 最大保存件数（20 / 50 / 100 / 200。デフォルト 50）
  • 履歴クリアボタン

パネル内操作

キー動作
↓ / Tab次へ
↑前へ
Return選択を貼り付け
Delete選択アイテムを削除
⌘Delete履歴を全クリア

必要な権限

アクセシビリティ（⌘V 送出のため）

画像は ~/Library/Application Support/FairyOperator/clipboardImages/ に PNG として保存される。

ウインドウ操作の妖精

ホットキー一発で最前面ウインドウを左右上下半分・最大化・別モニターへ。

主な機能

  • 左半分・右半分・上半分・下半分・最大化
  • 別スクリーンへの移動（サイズ比率を維持）
  • マルチモニター対応（ウインドウのあるスクリーンを自動判定）

ホットキー（デフォルト）

操作キー
左半分⌘⌃←
右半分⌘⌃→
上半分⌘⌃⇧↑
下半分⌃⌥↓
最大化⌃Return
別画面に移動⌃Space

必要な権限

アクセシビリティ

使用例

  • マウスドラッグせず、キーで素早くウインドウを左右分割配置
  • 外部モニターと内蔵モニター間の瞬時移動

IME 操作の妖精

指定キーに IME 切り替えアクションを割り当てるルールエンジン。

主な機能

  • 複数ルールの登録・並べ替え・削除
  • ルールごとの有効／無効
  • トリガーキー（修飾キー付きコンボ）
  • アクション 3 種：直接入力に切り替え／日本語入力に切り替え／トグル
  • 「通過」設定（元のキー動作も維持）
  • 対象アプリを bundleID で絞り込み（空なら全アプリ）

設定項目

  • 有効トグル
  • IME ルール一覧（追加・削除・有効/無効）
    • トリガーキー（「変更」ボタンでキャプチャ）
    • アクション選択
    • 通過チェックボックス
    • 対象アプリ（実行中アプリから追加、バッジ削除可）

デフォルトルール

トリガーアクション通過
Esc直接入力に切り替えON

必要な権限

アクセシビリティ

使用例

  • Vim 系エディタで Insert モードを抜けると同時に英数入力へ
  • 特定アプリでのみ Esc を IME OFF キーに

メニューバーをどける妖精

アプリメニューを一時的に隠してステータスアイコン領域を最大化。

主な機能

  • ステータスバーに常時 ◀／▶ トグルボタン
  • トグル ON: アプリメニューを空に差し替えてアイコン領域を確保
  • トグル OFF: 元のメインメニューを復元、前のアプリへフォーカス返却
  • フォーカスが離れると自動で閉じる
  • ホットキーでもトグル可能

設定項目

  • 有効トグル
  • 状態表示（開き／閉じ）
  • トグルホットキー（デフォルト: ⌃⌥M）

必要な権限

アクセシビリティ

使用例

  • ステータスアイコンが多くて埋もれているとき、アプリメニューを一時的に消して全アイコンを確認

Dock を飾り付ける妖精

Dock アイコン上にウインドウ数・重複識別ドット・応答なし砂時計などを表示。

主な機能

  • ウインドウ数バッジ — 2 つ以上ウインドウを持つアプリにカウント表示
  • 重複アプリ識別ドット — 同名アプリ複数起動時、各インスタンスを 8 色のドットで区別
  • Unity プロジェクト名 — Unity Editor のプロジェクト名をオーバーレイ
  • 応答なし表示 — フリーズしたアプリにオレンジの砂時計
  • アイコンクリック／ホバーでウインドウピッカーポップオーバー（サムネイル付き）
  • ピッカーからウインドウを直選択して前面化
  • マルチモニターで Dock が別画面に移動した際もオーバーレイが追従

設定項目

  • 有効トグル

必要な権限

アクセシビリティ／画面収録（サムネイル取得時）

使用例

  • 同アプリ複数インスタンスを Dock 上で識別
  • 多ウインドウアプリの Dock からの直接ウインドウ切り替え
  • 固まったアプリを一目で把握

ウインドウ ID 解決にプライベート API を使用しているため、将来の macOS アップデートで挙動が変わる可能性がある。

眠らない妖精

macOS のスリープを抑止。3 モードを切替可能で終了時に自動復元。

抑止モード

モード挙動権限
システムスリープのみ本体スリープ抑止（画面は消える）不要
システム＋ディスプレイ本体・ディスプレイ両方を抑止不要
蓋を閉じても継続pmset disablesleep 1 を適用管理者パスワード

主な機能

  • ホットキーでトグル
  • メニューバーメニューの「スリープしない」項目からもトグル
  • アプリ終了時に IOPMAssertion 解放・pmset 復元を保証
  • 蓋を閉じモードで管理者パスワードをキャンセルした場合は自動 OFF

設定項目

  • 有効トグル／状態表示（抑止中／停止）
  • 抑止対象モードの選択
  • トグルホットキー（デフォルト未設定）

蓋を閉じモードは ON／OFF の切り替えのたびに管理者パスワードが必要。アプリが強制終了された場合は pmset の復元が保証されない。

一般

アプリ全体の起動設定。

設定項目

  • ログイン時に起動（macOS のログイン項目に登録）
  • バージョン表示
