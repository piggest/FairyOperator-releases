Feature Guide

This page covers every feature (fairy) in FairyOperator — what it does, settings, hotkeys, and required permissions.

Requirements

Check the following environment and permissions before using FairyOperator.

Supported macOS13.0 (Ventura) or later
Launch modeLives in the menu bar. No Dock icon is shown.
Required permissionsAccessibility and Input Monitoring. Needed by most features.
Optional permissionsScreen Recording. Used when capturing window thumbnails.

First-time setup　Open System Settings → Privacy & Security, and add FairyOperator to Accessibility and Input Monitoring. Most features become available after that.

Key Remap Fairy

Remap the behavior of any key on a per-keyboard basis. Profiles are saved per device and switched automatically.

What it can do

  • Create and save an independent mapping profile per keyboard device.
  • Remap regular keys to other keys (via CGEvent tap).
  • Remap between modifier keys (e.g. CapsLock → Control).
  • Assign CapsLock to a regular key.
  • Remap HID-only keys that never surface as normal events.
  • Profiles for disconnected keyboards are kept and restored automatically on reconnect.
  • The most recently pressed key code is shown in real time in settings.

Settings

  • Master enable/disable toggle.
  • Keyboard list — connected and disconnected are shown separately; right-click to delete a profile.
  • Mapping editor — add/edit "from → to" pairs, toggle each one, and rename profiles.

Required permissions

Accessibility and Input Monitoring.

Examples

  • Reassign the JIS "Eisu" / "Kana" keys to Escape or another key.
  • Change CapsLock to Control for more comfortable editor navigation.

Synthesized events carry an internal marker (FAIRY) to prevent double-remapping. Modifier-vs-modifier remapping relies on macOS HID properties and may briefly persist after the app quits.

App Switcher Fairy

Press a hotkey to display a panel of recently used apps for quick switching — like Cmd+Tab, but more flexible.

What it can do

  • Show recently used apps in a grid up to 7 columns wide.
  • Hold the hotkey to move the selection forward/backward; release to confirm.
  • Press Esc to cancel; hover and click also select and confirm.
  • Right-click to open a window picker with thumbnails for picking among multiple windows.
  • Automatically restores minimized windows.
  • For Unity Editor, shows the per-project icon automatically.
  • Usage order is persisted across restarts.

Settings

  • Master enable/disable toggle.
  • Hotkeys for "Forward" and "Back".

Hotkeys

ActionDefault
Forward⌥Tab
Back⌥⇧Tab
CancelEsc
Open window pickerRight click

Required permissions

Accessibility (to restore minimized windows, etc.) and Screen Recording (for thumbnails).

Examples

  • Switch apps with fewer keystrokes than Cmd+Tab.
  • For apps with multiple windows, jump directly to a specific window.

Clipboard Fairy

Automatically captures copy operations into a history. Recall and paste past text or images with a hotkey.

What it can do

  • Polls the clipboard every 0.5s and adds new content to the history.
  • Saves both text and images (PNG / TIFF).
  • Automatically excludes sensitive types from password managers (org.nspasteboard.ConcealedType).
  • Pin items to keep them at the top and exempt from auto-deletion.
  • Incremental search to filter the history quickly.
  • Press number keys 1–9 to directly select the matching item.
  • On confirm, returns focus to the original app and sends ⌘V to paste.
  • Used items move to the top (just below pinned items).

Settings

  • Master enable/disable toggle.
  • Recall hotkey (default: ⌃V).
  • Search hotkey (default: ⌘F).
  • Max items (20 / 50 / 100 / 200, default 50).
  • Button to clear history.

Keys inside the panel

KeyAction
↓ / TabMove to next item.
↑Move to previous item.
ReturnPaste the selected item.

⌘DeleteClear all history.
EscClose the panel.

Required permissions

Accessibility (to send ⌘V).

Saved images are stored as PNGs under ~/Library/Application Support/FairyOperator/clipboardImages/.

Window Manager Fairy

Snap, maximize, or move the frontmost window between displays with a single hotkey.

What it can do

  • Snap the window to left half, right half, top half, bottom half, or maximize.
  • Move to another screen while preserving the relative size.
  • In multi-monitor setups, automatically detects which screen the window is currently on.

Hotkeys (defaults)

Required permissions

Accessibility.

Examples

  • Snap windows into left/right halves with the keyboard, no mouse dragging.
  • Instantly move a window between external and internal displays.

IME Control Fairy

A rule-based key-action engine for assigning IME (Japanese input) switching to keys you choose.

What it can do

  • Define multiple rules, reorder them, or delete them.
  • Enable or disable each rule individually.
  • Trigger keys can include modifiers.
  • Actions: "Switch to direct input", "Switch to Japanese input", or "Toggle".
  • With "Pass through" enabled, the trigger key's original behavior still fires — e.g. Escape can toggle IME and still act as Escape.
  • Limit each rule to specific apps by bundle ID. An empty list means all apps.

Settings

  • Master enable/disable toggle.
  • IME rule list. For each rule:
    • Trigger key (press the "Change" button to capture it).
    • Action.
    • "Pass through" checkbox.
    • Target apps (add from running apps; remove via badge).

Default rules

TriggerActionPass through
EscSwitch to direct inputON

Required permissions

Accessibility.

Examples

  • In Vim-style editors, automatically switch to English when leaving Insert mode.
  • Make Escape act as "turn IME off" only in selected apps.

Menu Bar Hider Fairy

Temporarily hides the app menu so more status icons fit on screen.

What it can do

  • Always shows a toggle arrow icon (◀ / ▶) in the menu bar.
  • Toggle ON: replaces the app menu with an empty menu, freeing room for status icons.
  • Toggle OFF: restores the original main menu and returns focus to the previous app.
  • If FairyOperator loses focus, the menu reverts automatically.
  • Hotkey can also drive the toggle.

Settings

  • Master enable/disable toggle.
  • Current state indicator ("Open" or "Closed").
  • Toggle hotkey (default: ⌃⌥M).

Required permissions

Accessibility.

Examples

  • When too many status icons are clipped, temporarily hide the app menu to reveal them.

Dock Decorator Fairy

Overlays information on Dock icons — window count, hang status, and more — to enhance Dock interactions.

What it can do

  • Window count badge　Shows a badge with the window count for apps with two or more windows.
  • Duplicate-app dots　For multiple instances of the same app, distinguishes them with 8-color dots.
  • Unity project name　Shows the current project name on the Unity Editor's Dock icon.
  • Not-responding indicator　Displays an orange hourglass on apps that have stopped responding.
  • Click or hover a Dock icon to open a popover window picker with thumbnails.
  • Pick a window directly from the picker to bring it to the front.
  • The overlay follows the Dock if it moves to another display.

Settings

  • Master enable/disable toggle.

Required permissions

Accessibility, and Screen Recording (for thumbnails).

Examples

  • Distinguish multiple instances of the same app on the Dock.
  • From the Dock, jump straight to a specific window of an app that has many.
  • Quickly spot an app that has stopped responding.

Window ID retrieval relies on private API, so behavior may change with future macOS updates.

Sleepless Fairy

Prevents macOS from sleeping. Three modes to choose from; the original state is restored automatically on exit.

Modes

ModeBehaviorPermission
System sleep onlyPrevents system sleep. The display may still turn off.None
System + displayPrevents both system and display sleep.None
Keep awake when lid closedApplies pmset disablesleep 1 so the laptop stays awake even when the lid is closed.Admin password

What it can do

  • Toggle prevention via hotkey.
  • Toggle from the "Stay Awake" item in the status bar menu.
  • On quit, always releases the IOPMAssertion and restores pmset.
  • If the admin password prompt is canceled in "Keep awake when lid closed" mode, automatically reverts to OFF.

Settings

  • Master enable/disable toggle. Shows current state ("Active" or "Stopped").
  • Target mode selector.
  • Toggle hotkey (unset by default).

"Keep awake when lid closed" requires the admin password every time you toggle it ON or OFF. If the app is force-quit, pmset restoration is not guaranteed.

General

Settings that apply to the whole app.

Settings

  • "Launch at Login" toggle — registers FairyOperator as a macOS login item.
  • Shows the current version.
